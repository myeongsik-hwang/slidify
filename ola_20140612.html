What is K-anonymity?

기본 용어 (1/2)

  • Attributes
    • (확실한) 식별자
    • SSN(Social Security Number), Address, Name 등
  • Quasi-Indetifiers
    • 불분명한 식별자
    • 우편번호, 생년월일, 성별 등

기본 용어 (2/2)

  • Sensitive Attributes
    • 개인과 관련된 민감 attributes
    • 질병질환 정보, 연봉 등
  • Equivalence class
    • 같은 익명의 데이터 레코드의 집합
    • 예를 들어, k-anonymity에서의 k의 대상은 equivalence class 내의 record 갯수를 의미함

  • Quasi-Indetifiers : Zip Code, Age

k-anonymity란?

Identity의 노출(즉, 개인식별 가능)을 방지하기 위해 각 equivalence class가 최소한 (al least) k개의 record로 구성하는 것

Attacks on k-anonymity

Homogeneity Attack on k-anonymity

Bob is a 27-year old man living in zip code 47678 and Bob’s record is in the table. So Bob corresponds to one of the first three records and must have heart disease.

Background Knowledge Attack

Carl is a 32-year old man living in zip code 47622. Therefore he is in the last equivalence class in below table. If you know that Carl has a low risk for heart disease then you can conclude that Carl probably has cancer.
